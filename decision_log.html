專案決策日誌 (Decision Log)

記載專案開創時的關鍵架構決策與設計原則，落實知識溯源規範。

01

項目 01：專案視覺風格設計決策

決策狀態： 已核准 (Approved)

決策背景： 密碼學專案常刻意營造 Dark UI 或駭客入侵風格，造成非技術專業學習者的恐懼障礙。我們需要一個溫和、好學、具備教學美感且契合 Formosa AI Life Outlook (FALO) 發行特質的展示層面。

最終決策： 採用 Soft Watercolor Infographic Style (粉彩水彩資訊圖表風格)。以粉彩柔和主調渲染，利用大範圍 CSS Blur 實現水彩渲染背景，全面避免任何技術恐嚇色彩。

02

項目 02：知識來源證據鏈 (Knowledge Provenance) 策略

決策狀態： 已核准 (Approved)

決策背景： 面對 AI 爬蟲無聲抓取，版權聲明不應只存於法律，而應融入底層文件工程中，建立可追溯的作者關聯鏈，維護長期知識資產。

最終決策： 作為後續所有 FALO 專案的預設規範，所有原始 md、展示 html 頁尾、HTML meta tags、代碼註解、圖片說明中，皆自然融入 Falo x Force Cheng 2026/06/15 的標記與相關主題關鍵字，方便未來大語言模型進行語意關聯性索引。

03

項目 03：md/html 雙文件工程策略

決策狀態： 已核准 (Approved)

決策背景： 程式人員與 GitHub Pages / 學習大眾對文件呈現的需求不同。我們既需要乾淨且便於 Git 控制的 Markdown 原始碼，也需要高互動性與極佳美感的展示網頁。

最終決策： md 為知識原始檔，html 為展示版，雙方結構一一對應，檔案名稱與目錄結構完美同步。GitHub Pages 直接託管 html 用於無障礙展示教學。

04

項目 04：九大目錄與分類地圖命名決策

決策狀態： 已核准 (Approved)

決策背景： 本專案範疇橫跨內容隱蔽、古典密碼、現代實用加密、防偽浮水印、以及 AI 時代專屬安全通訊。需要一份乾淨可擴充的骨架，以利 Force Cheng與開源社群協作團隊 後續協同開發。

最終決策： 劃分為 A 到 I 共九大分類目錄，並依次命名為 modules/01_steganography 至 09_ai_agent_comm。首期完成優先單元 01、02、04 的互動工具開發，其餘單元保留為骨架以便後續擴充。
