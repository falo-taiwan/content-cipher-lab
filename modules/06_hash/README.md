## 一、核心概念

**雜湊（Hashing）** 是指透過雜湊函數，將任意長度的輸入數據映射為固定長度的輸出值（稱為雜湊值、摘要或 Checksum）。

**⚠️ 核心關鍵觀念：雜湊絕對不等於加密 (Hash $\neq$ Encryption)**
* **可逆性**：加密是可逆的（透過解密還原明文）；而雜湊是**單向且不可逆**的。你無法從 MD5 或 SHA-256 的雜湊值反推出原始明文。
* **主要目的**：雜湊並不是為了「隱藏秘密」，而是為了**驗證完整性（Integrity）**，防止數據被篡改，或在不儲存使用者明文密碼的前提下進行密碼比對。

在 AI 與大型軟體工作流中，雜湊函數（例如 SHA256）被廣泛應用於軟體套件校驗、Git 歷史鏈驗證以及 AI 模型語料庫的重複數據刪除（De-duplication）。

⸻

## 二、研究與展示範疇

本單元主要整理與展示以下幾種雜湊與驗證技術：

1. **MD5 (Message-Digest Algorithm 5)**：早期極其常見的 128 位元雜湊演算法。儘管目前已被證實容易受到雜湊碰撞（Collision）攻擊而不適用於高安全場景，但它在快速比對檔案一致性（如下載檔案的 Checksum 驗證）上仍很實用。
2. **SHA 系列 (SHA1, SHA256, SHA512)**：美國國家安全局設計的安全雜湊演算法。目前 **SHA-256** 已成為密碼驗證、Git 提交與區塊鏈技術的業界標準。
3. **Checksum (校驗和)**：最簡單的資料完整性校驗方法（例如循環冗餘校驗 CRC32），用於網路傳輸封包的錯誤偵測。
