## 一、核心概念

**AI Agent 隱密通訊（AI Agent Secret Communication）** 指的是在多智能體系統（Multi-Agent Systems）中，Agent 之間使用人類不易察覺或難以理解的通道（Secret Channel）進行資訊交換的現象或技術。

這通常包括：
1. 使用特定欄位、Metadata（如 MCP 協定中的後台欄位）載入系統指令。
2. 開發「AI 原生密碼（AI Native Cipher）」，利用神經網路特定的激活區塊或高維向量編碼進行高隱蔽通訊。

這個單元反映了 AI 協同工作流（AI Workflow）中極為前沿的治理與稽核（AI Audit）課題：當 Agent 之間開始使用不受人類監管的秘密協議交談時，我們如何保證系統的透明性與安全性。

⸻

## 二、研究與展示範疇

本單元主要整理與展示以下幾種 Agent 通訊治理概念：

1. **Hidden Instructions (隱藏指令/間接注入)**：Agent 在讀取看似普通的電子郵件或網頁時，被隱藏在 HTML 註解或不可見 Unicode 字元中的惡意指令所控制。
2. **Agent Message & MCP Metadata**：在 Model Context Protocol (MCP) 或 Agent 與工具通訊的協定中，隱蔽傳遞敏感元數據。
3. **AI Native Cipher (AI 原生密碼學)**：探討藉由神經網路內建的高維數值表示法進行無文字的隱密語意傳遞。
