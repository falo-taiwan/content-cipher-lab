## 一、核心概念

**密碼書系統（Codebook Cipher）** 是人類歷史上極為重要的實用編碼保護技術。

它的運作核心是**語意代換**：
* 雙方事先秘密共享一本相同的「密碼對照表」（Codebook）。
* 這本密碼書中，每一個常見的「詞語」、「指令」或「人名」，都對應一個特定的「數字代號」或「無意義的英文字母組合」。
* 傳輸過程中，傳遞的內容完全是代號字串（例如 `8923 1045 7721`），即使被第三方截獲，若沒有該本密碼書，也完全無法推測真實意涵。

這與字元級別的古典密碼（如 Caesar、Vigenère）不同，密碼書是在**詞彙/語意級別**進行代換，這使得頻率分析攻擊變得極為困難。

## 二、研究與展示範疇

本單元主要整理與展示以下幾種密碼書相關概念：

1. **軍事密碼書與國共戰爭背景**：
   - 國共戰爭、二戰時期，軍事通信大量使用密碼書進行電報加密。
   - 例如將「部隊主力已抵達徐州」轉化為「`0541 3321 0089 9122`」。解碼端需逐字比對密碼本重組中文。
2. **外交密碼書**：歷史上各大國駐外使館與本國外交部通信所用的超大型密碼本，包含特定的地名與外交術語。
3. **商業代碼表（Commercial Codes）**：19 世紀末至 20 世紀初，為了節省昂貴的跨國海底電報（Telegraph）按字計費成本，企業使用代碼表將長句縮減為一個單字（如用 `ABACUS` 代表「本公司已收到款項並安排出貨」），意外地起到了基本混淆效果。
4. **一次性密碼本 (One-Time Pad, OTP) 概念**：
   - 密碼書的極致演進。
   - 使用完全隨機、與明文等長、且**僅使用一次**的密碼金鑰板（Pad）。
   - 在數學上已被證明是「無條件安全 / 絕對不可破譯」（Unbreakable）的，但其痛點在於金鑰的安全分發與管理成本極高。

## 三、教學重點與思考

1. **密碼書的致命缺陷**：
   - **實體安全性**：密碼書必須印製並派發。一旦其中一本密碼書落入敵手（被竊取或被策反），敵方即可解密所有歷史與未來截獲的通信，且發送端對此毫不知情。
   - **更新困難**：戰爭時期，頻繁更換密碼書是一項巨大的後勤挑戰。
2. **AI 時代的語意對照**：
   - 現代 LLM 本質上就是一個超大型的 Token 密碼書。LLM 將人類的詞彙編碼成數值型態的向量（Embedding Vector）。理解密碼書模式，能幫助我們以最直觀的方式理解現代 AI 內部如何將文字編碼為機器理解的代號，進而建立 AI 工作流（AI Workflow）的安全圍欄。
